The `chifu` CLI checks dependencies against known CVEs. Built for coding agents; works by hand or in CI.

## Install

```bash
npm i -g @marshell/chifu
# or
bun add -g @marshell/chifu
```

Zero install:

```bash
npx @marshell/chifu check
```

Standalone binaries (no Node): [GitHub releases](https://github.com/marshell-dev/chifu-cli/releases/latest).

Requires Node 18+ when installed from npm. One-command setup: [Quickstart](/quickstart).

## `chifu check`

```bash
chifu check [path]
```

`[path]` defaults to cwd. Reads manifest + lockfile (or `node_modules`) and resolves the full tree.

### Options

| Flag | What it does |
| --- | --- |
| `--json` | Machine-readable JSON (what the agent skill reads) |
| `--verbose` | List every CVE instead of per-package rollup |
| `--fail-on-findings` | Exit non-zero when vulnerabilities are found (CI) |
| `--fail-on-install-scripts` | Exit non-zero when new/changed deps have install lifecycle scripts |
| `--all-install-scripts` | Scan full lockfile for scripts (default: git diff only) |
| `--api-url <url>` | Override backend origin |
| `--no-update` | Skip auto-update check |

`check` flags packages with `preinstall` / `install` / `postinstall` / `prepare` in new or changed lockfile deps.

### Output

Collapsed view (default):

```text
  3 of 143 dependencies are vulnerable

  CRITICAL  lodash    4.17.4  → 4.18.0    (8 CVEs)
  HIGH      axios     0.21.0  → 0.32.0    (13 CVEs)
```

Clean:

```text
  ✓ No known vulnerabilities in 143 dependencies.
```

`→ no fix yet` when no patched version exists.

### JSON shape

```json
{
  "ecosystem": "npm",
  "scanned": 143,
  "vulnerablePackages": 3,
  "packages": [
    {
      "name": "lodash",
      "version": "4.17.4",
      "recommendedVersion": "4.18.0",
      "worstSeverity": "critical",
      "cveCount": 8
    }
  ],
  "findings": [
    {
      "name": "lodash",
      "cve": "CVE-2021-23337",
      "severity": "high",
      "fixedVersion": "4.17.21",
      "advisoryUrl": "https://github.com/advisories/..."
    }
  ]
}
```

`packages` = actionable rollup. `findings` = per-CVE detail.

### Exit codes

| Code | Meaning |
| --- | --- |
| `0` | Success (findings OK unless `--fail-on-findings`) |
| `1` | Vulnerabilities found with `--fail-on-findings` |
| `2` | Error (no manifest, network failure, etc.) |

## `chifu fix`

Apply recommended **npm** fixes (version bumps + `overrides` for transitives).

```bash
chifu fix [path]
```

| Flag | What it does |
| --- | --- |
| `--json` | Machine-readable fix report |
| `--dry-run` | Show planned changes without writing files |
| `--api-url <url>` | Override backend origin |
| `--no-update` | Skip auto-update check |

Re-runs check after applying fixes. For breaking majors, prefer the agent skill over blind `fix`.

## `chifu login`

```bash
chifu login              # browser pairing
chifu login chf_xxx      # save key directly (CI)
```

Required before `chifu check`. Results sync to your [dashboard](/dashboard).

## `chifu logout`

```bash
chifu logout
```

Clears the saved API key from local config.

## Configuration

| Platform | Config path |
| --- | --- |
| macOS / Linux | `~/.config/chifu/config.json` |
| Windows | `%APPDATA%\chifu\config.json` |

| Variable | Default | Purpose |
| --- | --- | --- |
| `CHIFU_API_KEY` | — | API key without `login` |
| `CHIFU_API_URL` | `https://api.marshell.dev` | Backend origin |
| `CHIFU_WEB_URL` | `https://marshell.dev` | Dashboard origin (login flow) |
| `CHIFU_NO_UPDATE` | — | Set to `1` to disable auto-update |
| `CHIFU_UPDATE_INTERVAL` | `0` | Min seconds between update checks |

Key resolution: `CHIFU_API_KEY` → config file.

## Global flags

| Flag | What it does |
| --- | --- |
| `chifu --help`, `-h` | Usage |
| `chifu --version`, `-v` | Version |
