Most people never touch the API — the dashboard does everything. But if you want to
trigger scans from CI, a script, or your own tooling, chifu speaks a small REST API
authenticated with an API key.

## Your one-time key

Right after you sign up, chifu shows your API key **once**, as a banner on the
dashboard. This is the only time the full key is displayed.

<Warning>
  Copy it immediately and store it somewhere safe (a password manager or your CI
  secrets). chifu only keeps a hash — if you lose it, you can't recover it, you'd need a
  new one.
</Warning>

If you only use the dashboard, you can ignore the key entirely.

## Create more keys

Need another key — for the [MCP server](/mcp), CI, or a second machine? Open
**Settings → API keys**, click **Create key**, give it a name, and copy it. Like
the signup key, the full value is shown **once**. Revoke any key from the same
screen when you're done with it.

## Authenticating

Send the key as a Bearer token on every request:

```bash
curl https://api.marshell.dev/api/v1/scan \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"target":"https://myapp.com"}'
```

The response includes a `scanId` you can use to fetch the result:

```bash
curl https://api.marshell.dev/api/v1/scan/SCAN_ID \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Note>
  You can only scan domains you've [verified](/domains), and API scans count against
  your monthly [quota](/plans) just like dashboard scans.
</Note>

## Endpoints

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/api/v1/scan` | Enqueue a scan for a target. Returns `{ scanId }`. |
| `GET` | `/api/v1/scan/{scanId}` | Fetch a scan's full result. |

## Keeping it safe

- Never commit the key to a repo or paste it in screenshots.
- Store it in your platform's secrets (Vercel/GitHub Actions environment variables).
- Treat it like a password — anyone with it can run scans as you.

<Note>
  Programmatic API access is part of the **Pro** plan. See [Plans](/plans).
</Note>
