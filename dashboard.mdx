chifu runs fully offline of an account. `chifu check` reads your files, checks
them against the vulnerability database, and prints the result, no sign-in
required. Signing in adds one thing: your results sync to the
[dashboard](https://marshell.dev/dashboard) so you have a history across
projects.

## Why sign in

A check that synced shows `"synced": true` in the JSON output. When you are not
signed in, the CLI reminds you that results were not saved.

## Sign in

The simplest way is from the CLI:

```bash
chifu login
```

This opens your browser, pairs the device, and saves your key automatically. No
copy-paste of secrets.

You can also pass a key directly:

```bash
chifu login chf_xxxxxxxx
```

The [installer](/quickstart) can sign you in during setup, or you can pass
`--api-key chf_xxxxxxxx` to it.

## API keys

API keys start with `chf_`. You get your first key when you sign up, and you can
view and manage keys from the dashboard.

A key is read in this order: the `CHIFU_API_KEY` environment variable, then the
[config file](/cli#configuration). `chifu check` requires one. Use the
environment variable in CI:

```bash
CHIFU_API_KEY=chf_xxxxxxxx chifu check
```

<Warning>
  Treat your API key like a password. It is stored locally in a file readable
  only by you. If a key is exposed, revoke it from the dashboard and run
  `chifu login` again.
</Warning>
