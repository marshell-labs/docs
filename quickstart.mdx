The installer adds the `chifu` CLI and drops the `chifu-dep-guard` skill into every coding agent it finds.

```bash
curl -fsSL https://marshell.dev/install.sh | sh
```

```bash
npx @marshell/chifu-wizard@latest
```

<AccordionGroup>
  <Accordion title="Windows">
    ```powershell
    irm https://marshell.dev/install.ps1 | iex
    ```
  </Accordion>
</AccordionGroup>

The wizard does three things:

1. **Installs the CLI** — `chifu` on your PATH (or use `npx @marshell/chifu` with no global install).
2. **Wires up your agents** — writes the skill into Claude Code, Cursor, Windsurf, Codex, and more. See [Supported agents](/agents).
3. **Signs you in** — browser pairing mints your `chf_` key. `chifu check` requires an account.

Prefer manual setup? See [CLI](/cli) and [Agent skill](/skill).

## Run your first check

From any project with a lockfile:

```bash
chifu check
```

Clean:

```text
  ✓ No known vulnerabilities in 143 dependencies.
```

Findings:

```text
  3 of 143 dependencies are vulnerable

  CRITICAL  lodash    4.17.4  → 4.18.0    (8 CVEs)
  HIGH      axios     0.21.0  → 0.32.0    (13 CVEs)
  MEDIUM    minimist  1.2.0   → 1.2.3     (1 CVE)
```

Each row is one package with the single upgrade that clears all its CVEs.

## Let your agent take over

Once the skill is installed, your agent runs `chifu check` when it changes dependencies or before it commits. Ask it to add a dependency and watch it check and fix before finishing.
