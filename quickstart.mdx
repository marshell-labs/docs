chifu installs with a single command. The installer adds the `chifu` CLI and
drops the `chifu-dep-guard` skill into every coding agent it finds on your
machine.

## Install

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl -fsSL https://marshell.dev/install.sh | sh
    ```
  </Tab>
  <Tab title="Windows">
    ```powershell
    irm https://marshell.dev/install.ps1 | iex
    ```
  </Tab>
  <Tab title="npx / bunx">
    ```bash
    npx @marshell/chifu-wizard@latest
    # or
    bunx @marshell/chifu-wizard@latest
    ```
  </Tab>
</Tabs>

The installer walks you through three things:

<Steps>
  <Step title="Install the CLI">
    It installs `@marshell/chifu` globally so `chifu` is on your PATH. If a
    global install is not possible, your agent can still run it with `bunx` or
    `npx`.
  </Step>
  <Step title="Wire up your agents">
    It detects Claude Code, Cursor, Windsurf, Codex, OpenCode, Gemini CLI, and
    Cline, then writes the skill into each one in its native format. See
    [supported agents](/agents).
  </Step>
  <Step title="Sign in (required)">
    chifu needs an account. Sign in to mint your `chf_` key — `chifu check`
    won't run without it. Browser pairing, no copy-pasting secrets.
  </Step>
</Steps>

<Tip>
  Prefer to install the pieces yourself? See [the CLI](/cli) and
  [the skill](/skill) for manual setup.
</Tip>

## Run your first check

From any npm project, run:

```bash
chifu check
```

A clean project looks like this:

```text
  ✓ No known vulnerabilities in 143 dependencies.
```

A project with vulnerable packages looks like this:

```text
  3 of 143 dependencies are vulnerable

  CRITICAL  lodash    4.17.4  → 4.18.0    (8 CVEs)
  HIGH      axios     0.21.0  → 0.32.0    (13 CVEs)
  MEDIUM    minimist  1.2.0   → 1.2.3     (1 CVE)

  Run with --verbose to list individual CVEs.
```

Each row shows the package, its current version, the single upgrade that clears
all of its CVEs, and how many CVEs that is.

<Note>
  `chifu check` requires an account — run `chifu login` first (or set
  `CHIFU_API_KEY`). It reads your `package.json` and lockfile, checks them
  against the vulnerability database, and prints the result.
</Note>

## Let your agent take over

You do not have to run `chifu` by hand. Once the skill is installed, just keep
building. The next time your agent adds or upgrades a dependency, or right
before it commits, it runs `chifu check` on its own and fixes whatever it finds.

Try it: ask your agent to add a dependency, then watch it check and clean up
before finishing.
