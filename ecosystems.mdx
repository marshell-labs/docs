---
title: Ecosystems
description: Package managers and lockfiles chifu supports across eight ecosystems.
---

`chifu check` auto-detects **one ecosystem per run** from the first manifest with resolved dependencies.

Detection order: npm → PyPI → Maven → Go → NuGet → crates.io → Packagist → RubyGems.

## Supported ecosystems

| Ecosystem | Manifests / lockfiles |
| --- | --- |
| **npm** | `package-lock.json`, `npm-shrinkwrap.json`, `pnpm-lock.yaml`, `yarn.lock`, `bun.lock`, `package.json`, `node_modules` |
| **PyPI** | `requirements.txt`, `poetry.lock`, `Pipfile.lock`, `uv.lock`, `pylock.toml`, `pyproject.toml` |
| **Maven / Gradle** | `pom.xml`, `build.gradle`, `build.gradle.kts`, Gradle lockfiles |
| **Go** | `go.mod`, `go.sum` |
| **NuGet** | `packages.lock.json`, `packages.config`, `.csproj` |
| **crates.io** | `Cargo.lock`, `Cargo.toml` |
| **Packagist** | `composer.lock`, `composer.json` |
| **RubyGems** | `Gemfile.lock`, `gems.locked`, `Gemfile` |

## Monorepos

Mixed stacks in one repo? `cd` into each subproject and run `chifu check` separately. The skill tells agents to use the directory with the relevant lockfile — often `frontend/`, not the git root.

## Fix support

| Ecosystem | `chifu fix` | Agent fix loop |
| --- | --- | --- |
| npm | Yes | Yes |
| All others | No | Yes — edit manifest, re-check |

npm examples in [How it works](/how-it-works) use `overrides` and `npm install --package-lock-only --ignore-scripts`. Other ecosystems follow the fix recipes in the [agent skill](/skill).

## Examples

```bash
chifu check                    # cwd
chifu check ./services/api     # subproject
chifu check --json             # agent output
```
