`chifu-dep-guard` is installed by the [wizard](/quickstart) into your coding agent's native format. It tells the agent **when** to run `chifu check --json` and **how** to fix findings without executing untrusted package code.

Full agent reference: [SKILL.md on GitHub](https://github.com/marshell-dev/chifu-skill/blob/main/SKILL.md).

## When the agent runs chifu

| Trigger | Action |
| --- | --- |
| User says commit / push / PR | Check first; fix loop if needed |
| Agent adds, upgrades, or pins a dep | Check after manifest edit |
| Task touched a lockfile | Check before ending the turn |
| User asks audit / CVE / supply chain | Check + fix loop |

**Never commit, push, or open a PR until `vulnerablePackages` is 0** (or only unfixable entries remain and the user was told).

## Fix loop (summary)

1. `chifu check --json` from the project root with the lockfile
2. Read `packages[]` — bump to `recommendedVersion`, critical-first
3. Re-resolve npm lockfile only: `npm install --package-lock-only --ignore-scripts`
4. Re-check until clean
5. User runs real install + tests before commit

## The safety rule

**No full install during the fix loop.** No `npm install`, `npm ci`, `yarn`, `pnpm install`, or `bun install`. Lifecycle scripts on compromised packages are the attack surface chifu avoids.

## Commands the agent uses

```bash
chifu check --json
bunx @marshell/chifu@latest check --json   # fallback
```
