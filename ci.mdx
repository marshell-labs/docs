The agent skill is the first line of defense. CI is the backstop — catches hand-added deps and anything that slips through.

## Gate a build

```bash
chifu check --fail-on-findings
```

| Exit code | Meaning |
| --- | --- |
| `0` | No vulnerabilities |
| `1` | Vulnerabilities found (with `--fail-on-findings`) |
| `2` | Error |

Without `--fail-on-findings`, findings still exit `0`. Always use the flag in CI.

## Install-script gate (optional)

Flag new or changed deps that run install lifecycle scripts:

```bash
chifu check --fail-on-findings --fail-on-install-scripts
```

Scan the full lockfile instead of git diff only:

```bash
chifu check --fail-on-findings --all-install-scripts
```

## GitHub Actions

```yaml
name: chifu
on: [push, pull_request]

jobs:
  dependencies:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - name: Check dependencies
        env:
          CHIFU_API_KEY: ${{ secrets.CHIFU_API_KEY }}
        run: npx @marshell/chifu@latest check --fail-on-findings
```

chifu reads your committed lockfile — **no install step required**. That keeps CI fast and avoids running package lifecycle scripts.

Set `CHIFU_API_KEY` from repo secrets. Every check requires an account.

## Other CI systems

```yaml
# GitLab CI example
chifu:
  image: node:20
  script:
    - npx @marshell/chifu@latest check --fail-on-findings
  variables:
    CHIFU_API_KEY: $CHIFU_API_KEY
```
