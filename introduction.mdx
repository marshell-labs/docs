chifu is dependency security for AI coding agents. Run `chifu check` in any
repo and it matches every package, direct and transitive, against 65,000+
known vulnerabilities (NVD + OSV). Your AI coding agent fixes the vulnerable
ones before you commit.

It is two small pieces that work together:

<CardGroup cols={2}>
  <Card title="A CLI" icon="terminal">
    `chifu check` reads your dependency tree and reports which packages have
    known CVEs, plus the single upgrade that clears each one. It detects. It
    never installs anything.
  </Card>
  <Card title="An agent skill" icon="robot">
    `chifu-dep-guard` teaches your coding agent when to run the check and how
    to fix the findings safely, without ever executing untrusted package code.
  </Card>
</CardGroup>

<Note>
  There is no MCP server to run. chifu is a CLI plus an agent skill, wired into
  your editor by a one-command installer.
</Note>

## The problem

AI coding agents add, bump, and pin dependencies constantly, and some of those
versions ship with known vulnerabilities. The agent moves on, and the CVE lands
in your repo. You find out weeks later in a Dependabot pull request, or you
don't find out at all.

chifu keeps that check inside the coding loop. The moment your agent touches a
dependency, it can catch the vulnerable version and fix it before the change is
ever committed.

## How it works

<Steps>
  <Step title="Install once">
    One command installs the CLI and drops the `chifu-dep-guard` skill into
    every coding agent you use.
  </Step>
  <Step title="Your agent checks">
    Whenever the agent changes a dependency (or before a commit), it runs
    `chifu check` and gets back the vulnerable packages with their fixes.
  </Step>
  <Step title="Your agent fixes">
    The agent upgrades the vulnerable packages, handles any breaking changes,
    and re-checks until the project is clean.
  </Step>
</Steps>

Read [how it works](/how-it-works) for the full detect-and-fix loop, including
the rule that makes it safe.
